# 1. ОПИСАНИЕ ПРОЕКТА

Проект — **высоконагруженный новостной / деловой медиа‑портал**, объединяющий лучшие практики трёх эталонных ресурсов:

| Источник | Что берём за основу |
|----------|---------------------|
| **WSJ.com** | Премиальная подача, платная подписка (metered paywall), система членства (WSJ+), рубрикатор делового СМИ, качественная типографика, newsletters, подкасты, персонализация. |
| **FoxBusiness.com** | Живые биржевые тикеры (котировки, крипта, счётчик госдолга), видео‑шоу, watchlist, «горячие темы», плотная рекламная монетизация, market data. |
| **Vaqt.uz** | Мультиязычность (uz/ru/en/…), виджеты курсов валют и крипты, компактная лента, локальный деловой контекст, витрина «Выбор редактора», мобильные приложения, соц‑интеграции. |

**Тип продукта:** контент‑платформа (Publisher Platform) класса «цифровое СМИ».

**Ключевые характеристики:**
- Публикация статей, новостей, лонгридов, видео, подкастов, фотогалерей, инфографики, live‑блогов.
- Рубрикатор: Экономика, Рынки, Бизнес, Финансы, Технологии, Недвижимость, Общество, Мир, Спорт, Lifestyle, Мнения (Opinion).
- Финансовые виджеты: котировки акций, индексы, крипта, курсы валют, watchlist, счётчик показателей.
- Монетизация: подписка (paywall), реклама (собственный ad‑server + внешние сети), нативная реклама, спонсорский контент.
- Мультиязычность и мультирегиональность.
- Мощная CMS для редакции + полноценная система управления рекламой.
- Высокие требования к SEO, скорости (Core Web Vitals), доступности (WCAG 2.2 AA).

# 2. ЦЕЛИ И ЗАДАЧИ

**Бизнес‑цели:**
1. Создать масштабируемое цифровое СМИ с несколькими потоками монетизации (подписка + реклама + партнёрский контент).
2. Обеспечить редакции удобный инструмент публикации без участия разработчиков.
3. Дать владельцу полный самостоятельный контроль над рекламой (собственный ad‑server).
4. Максимизировать органический трафик за счёт эталонного SEO и скорости.
5. Удержание аудитории через персонализацию, рассылки, push, подкасты, watchlist.

**Продуктовые цели:**
- Time‑to‑first‑byte < 200 мс (кэш/CDN), LCP < 2.5 с, INP < 200 мс, CLS < 0.1.
- Paywall с гибкими правилами (metered / freemium / hard).
- Конверсия «читатель → подписчик» и «аноним → зарегистрированный».
- Доступность контента на всех устройствах (mobile‑first).

**Технические цели:**
- Headless‑архитектура (разделение CMS/бэкенда и фронтенда), SSR/ISR для SEO.
- Горизонтальное масштабирование, отказоустойчивость, наблюдаемость (observability).
- Безопасность уровня enterprise (OWASP, GDPR, PCI‑DSS для платежей).

# 3. АНАЛИЗ ЭТАЛОННЫХ САЙТОВ (ЭТАП 1)

## 3.1. Общая ДНК всех трёх сайтов

Все три ресурса — **иерархические контент‑порталы** со схожей архитектурой информации:

```
Хедер (лого + навигация по рубрикам + поиск + вход/подписка + доп. виджеты)
    ↓
Тикер/строка (котировки, горячие темы, breaking news)
    ↓
Главная (герой‑блок + сетка лент по рубрикам + сайдбар‑виджеты + реклама)
    ↓
Страница рубрики (лента статей + фильтры + пагинация + реклама)
    ↓
Страница статьи (заголовок + медиа + тело + paywall + related + реклама + комментарии)
    ↓
Футер (карта сайта + соц + приложения + правовые + подписка)
```

## 3.2. WSJ.com — детальный разбор

**Назначение:** премиальное деловое издание с платным доступом.

**Хедер:**
- Логотип WSJ по центру (классическая газетная верстка) либо слева.
- Верхняя утилити‑панель: дата, издание/регион, «Print Edition», «Video», «Podcasts», «Latest Headlines».
- Основная навигация (рубрики): World, U.S., Politics, Economy, Business, Tech, Markets, Opinion, Personal Finance, Life & Work, Style, Sports, Arts, Real Estate.
- Кнопки: **Sign In** и **Subscribe** (акцентная).
- Иконка поиска (раскрывающаяся строка).
- Мега‑меню: при наведении на рубрику выпадает панель с подрубриками и подборкой топ‑материалов.

**Главная:**
- Крупный герой‑материал (lead story) с фото/видео.
- Колоночная газетная сетка: главная колонка + правая колонка «What's News» (дайджест коротких новостей) + левая колонка Opinion.
- Блоки по рубрикам с 3–5 материалами каждый.
- Врезки: Markets snapshot (индексы DJIA, S&P, Nasdaq), «Most Popular», «Recommended for you» (персонализация).
- Рекламные слоты (leaderboard, MPU 300×250, in‑feed native).

**Paywall (ключевая механика):**
- Metered / freemium: часть материалов бесплатна, премиальные — заблокированы (значок «замок»).
- При достижении лимита показывается **paywall‑модалка / inline‑блок**: «Subscribe to continue», варианты подписки, «Already a member? Sign in».
- Реализация: серверная проверка права доступа + client‑side счётчик (cookie/localStorage) + entitlement API. Контент премиальных статей не отдаётся целиком в HTML для незалогиненного (защита от обхода через «reader mode»).

**Membership / WSJ+:** личный кабинет с эксклюзивными предложениями, событиями, рассылками, подкастами, разделом WSJ Students, digital‑журнал Noted (18–34).

**Личный кабинет:** управление подпиской, платёжные данные, рассылки, сохранённые статьи (Saved), watchlist, настройки уведомлений.

**Динамика/JS:** ленивая подгрузка лент (infinite scroll на рубриках), live‑обновление котировок (WebSocket/polling), персональные рекомендации (recommendation API), A/B тесты paywall.

## 3.3. FoxBusiness.com — детальный разбор

**Назначение:** массовое деловое СМИ с сильным видео и биржевыми данными.

**Хедер:**
- Логотип Fox Business.
- Основная навигация: Personal Finance, Economy, Markets, Watchlist, Lifestyle, Real Estate, Tech, Video, Podcasts.
- Мега‑меню с подрубриками (напр., Markets → Stocks, US Markets, Cryptocurrency, Futures & Commodities, Bonds, ETFs, Business Leaders, Industries, Retail, Pre‑Markets, Mutual Funds).
- **Строка «Hot Topics»**: U.S. Stocks, Product Recalls, Trump News, AI, Weather.
- Кнопки: Login, Watch Live TV, иконка поиска.

**Тикеры и market data:**
- Живой тикер котировок (акции, индексы), крипта, **счётчик национального долга** (реального времени, инкрементальный JS‑счётчик).
- Данные от провайдера (FactSet), пометка «delayed by at least 15 minutes».
- Watchlist — персональное отслеживание бумаг (требует логина).

**Главная:**
- Герой‑материал (double‑width feature card).
- Ленты‑карточки: заголовок + миниатюра + категория + опц. описание.
- **Видео‑блоки**: карточки с play‑overlay, сгруппированы по шоу (The Bottom Line, Kudlow, Big Money Show).
- Тематические блоки: Money, Lifestyle, Real Estate, Tech, Sports, Small Business, Luxury Properties, Rich & Famous.
- Плотная реклама (leaderboard, MPU, sticky bottom, in‑content, video pre‑roll).

**Футер:** правовые ссылки, Channel Finder, Jobs, Media Relations, кросс‑промо (Fox News, Fox Nation, Fox Business Go), соц‑сети (FB, X, Instagram, YouTube, Flipboard, LinkedIn, RSS, Spotify, iHeartRadio), дисклеймер о правах.

**Динамика/JS:** live котировки (WebSocket/poll), счётчик долга (клиентский инкремент от базового значения × ставка/сек), видео‑плеер (HLS), lazy‑load, бесконечная лента.

## 3.4. Vaqt.uz — детальный разбор

**Назначение:** локальное деловое/новостное издание Узбекистана, мультиязычное.

**Хедер:**
- Логотип.
- Навигация: Oʻzbekiston (Узбекистан), Dunyo (Мир), Iqtisodiyot (Экономика), Moliya (Финансы), Biznes (Бизнес), Jamiyat (Общество), + «Yana» (Ещё) → InfoTexnologiya, Sport.
- **Переключатель языка** (uz/ru/en/…), заложен в URL (`/uz`, `/ru`).
- Поиск.

**Главная:**
- Герой‑история с таймстампом и заголовком.
- Хронологическая лента карточек: время + дата + заголовок + миниатюра.
- **Сайдбар‑виджеты**:
  - Курсы валют (¥, $, €, £, ₽, ₣) с % изменения.
  - «Muharrir tanlovi» (Выбор редактора) — карусель.
  - Фондовый рынок Республики, «Kripto olam» (крипта).
  - Фото‑ и видео‑контент.

**Футер:** контакты, соц‑сети (Instagram, Telegram, YouTube, LinkedIn, X, TikTok, Facebook, Threads), кнопки скачивания мобильных приложений, правовые сведения.

**Динамика/JS:** обновление курсов валют/крипты (API polling), карусель «Выбор редактора», ленивая подгрузка ленты, мультиязычная маршрутизация.

## 3.5. Синтез: что реализуем в нашем проекте

Объединяем: **рубрикатор и paywall/membership WSJ** + **market data, video, watchlist, плотную рекламу Fox Business** + **мультиязычность, валютные/крипто‑виджеты, мобильные приложения, соц‑интеграции Vaqt.uz**, добавляя собственную полноценную CMS и рекламный сервер.

## 5.2. Связи между разделами

- **Главная** агрегирует материалы из всех рубрик (кураторские + алгоритмические блоки).
- **Рубрика → Подрубрика → Статья**: иерархия «section → subsection → article»; статья принадлежит одной основной рубрике + N тегам/темам.
- **Статья ↔ Автор**: many‑to‑many (соавторы), у автора — страница со всеми материалами.
- **Статья ↔ Тег/Тема**: many‑to‑many; тема агрегирует материалы разных рубрик (кросс‑секционные сюжеты).
- **Статья → Related**: связанные материалы (по тегам/рубрике/ML‑рекомендации).
- **Markets ↔ Quote ↔ Watchlist**: инструмент упоминается в статьях (тикер‑линки), добавляется в watchlist.
- **Paywall ↔ Аккаунт ↔ Подписка**: entitlement‑проверка на каждой premium‑странице.
- **Реклама** внедряется во все ленты и статьи через слоты (ad‑slots), управляемые из CMS.

## 6.0. Глобальные компоненты (на всех страницах)

**Header (шапка):**
- Блоки: логотип; основная навигация (мега‑меню); утилити (дата/регион, поиск, язык, вход/аккаунт, «Подписаться», Live TV); строка тикеров/breaking.

# 8. АДАПТИВНАЯ / МОБИЛЬНАЯ ВЕРСИЯ (ЭТАП 4)

**Подход:** mobile‑first. Брейкпоинты: 320 / 360 / 375 / 390 / 414 / 768 / 1024 / 1280+.

## 8.1. Общие мобильные принципы

- Одноколоночная лента; карточки full‑width со стекингом «изображение → kicker → заголовок → мета».
- Хедер сжат: бургер слева, лого по центру, поиск/аккаунт справа; тикеры — горизонтальный скролл.
- **Нижнее таб‑меню (bottom nav)** на телефонах: Главная, Рубрики, Поиск, Watchlist/Рынки, Аккаунт — sticky, скрывается при скролле вниз.
- Крупные тап‑таргеты (≥ 44×44 px), увеличенные отступы.
- Sticky‑bottom рекламный баннер (закрываемый), не перекрывающий bottom‑nav (учитывать safe‑area iOS).

## 8.2. Поведение по разрешениям

**320 px (мелкие Android/старые):**
- Одна колонка, изображения 16:9 full‑width. Заголовки 20–22 px. Бургер‑меню полноэкранное (drawer). Фильтры → bottom‑sheet. Таблицы (котировки, тарифы) → горизонтальный скролл в контейнере с fade‑краями. Мега‑меню → аккордеон. Поиск → полноэкранный overlay. Popup/paywall → полноэкранный лист. Кнопки full‑width.

**360 px (типовой Android):** как 320, чуть больше воздуха; карточки с миниатюрой 40%/текст 60% (horizontal card) допустимы.

**375 px (iPhone SE/mini):** учёт safe‑area (notch); bottom‑nav + safe‑area‑inset‑bottom; sticky‑ad над bottom‑nav.

**390 px (iPhone 12–15):** базовый мобильный таргет; карусели со свайпом (snap‑scroll), индикаторы‑точки; галереи — свайп.

**414 px (Plus/Max):** двухколоночная сетка карточек в некоторых блоках (напр. «Most popular»/видео) при ширине ≥ 400.

**768 px (планшет портрет):** 2 колонки; сайдбар возвращается частично; бургер сохраняется либо частичная горизонтальная навигация; фильтры — горизонтальная панель; таблицы помещаются; bottom‑nav скрывается (появляется полная навигация).

**1024 px (планшет ландшафт / малый ноут):** 3‑колоночная сетка лент; полноценная горизонтальная навигация + мега‑меню (tap→click); сайдбар‑виджеты; MPU в сайдбаре.

**1280+ px (desktop):** полная газетная сетка, все сайдбары, hover‑эффекты, leaderboard 970×250.

## 8.3. Мобильные паттерны

- **Бургер‑меню:** slide‑in drawer с рубриками (аккордеон подрубрик), языком, входом/подпиской, соц‑сетями.
- **Фильтры/сортировки:** bottom‑sheet с «Применить/Сбросить».
- **Popup/paywall:** полноэкранный/bottom‑sheet, закрываемый; не блокирует скролл фона агрессивно.
- **Свайпы:** карусели, галереи, «следующая статья»; свайп‑to‑save опц.
- **Горизонтальный скролл:** тикеры, чипы рубрик, таблицы котировок/тарифов (обёрнуты в `overflow-x:auto`).
- **Sticky:** уменьшенный хедер при скролле вверх, bottom‑nav, reading‑progress, sticky‑ad.
- **Pull‑to‑refresh** на ленте (PWA).

## 8.4. Улучшения относительно оригиналов (мобайл)

- Единый **bottom‑nav** (у оригиналов его нет — навигация удобнее).
- **Оффлайн‑чтение** сохранённых статей (PWA + service worker).
- **Режим чтения** (крупный шрифт, тёмная тема, TTS «Слушать статью»).
- Ненавязчивый paywall (bottom‑sheet вместо полноэкранного блокера).
- Экономия трафика: адаптивные изображения (AVIF/WebP, `srcset`), lazy‑load, отложенная загрузка рекламы.

# 10. РЕКЛАМНАЯ СИСТЕМА (ЭТАП 6)

Собственный **Ad‑Server** внутри CMS + подключение внешних сетей. Владелец полностью самостоятельно управляет рекламой.

## 10.1. Архитектура

```
Ad Manager (CMS)
├── Рекламодатели (Advertisers) → Кампании (Campaigns) → Объявления (Creatives)
├── Рекламные места (Ad Slots / Zones) — привязка к позициям на страницах
├── Правила показа (Targeting + Scheduling + Frequency + Priority/Weight)
├── Ротация и приоритизация (weighted rotation / waterfall / header bidding)
├── Трекинг (impressions, viewability, clicks, conversions) → Статистика
└── Внешние сети (GAM/AdSense, прямые сделки, programmatic) через единый слой
```

**Слой доставки:** frontend рендерит `AdSlot` по коду зоны → запрос к ad‑decision API → сервер выбирает объявление по таргетингу/приоритету/частоте → отдаёт creative + пиксели трекинга. Ленивая загрузка (IntersectionObserver), lazy‑render, reserve‑space (против CLS).

## 10.2. Типы рекламных блоков

| Тип | Место | Размеры (desktop) | Адаптив |
|-----|-------|-------------------|---------|
| **Leaderboard / Billboard** | верх страницы | 728×90, 970×90, 970×250 | 320×50/320×100 моб. |
| **MPU / Medium Rectangle** | сайдбар, in‑content | 300×250, 300×600 (half‑page) | 300×250 моб. |
| **In‑feed native** | внутри ленты | адаптивная карточка | full‑width |
| **In‑content** | между абзацами | 300×250 / fluid | адаптив |
| **Sticky bottom (anchor)** | низ экрана | 728×90 / 320×50 | закрываемый |
| **Sticky side rail** | боковые поля | 160×600 | скрыт на моб. |
| **Popup / Interstitial** | модалка/переход | full/оверлей | full‑screen |
| **Fullscreen / Welcome** | вход на сайт | full | full |
| **Carousel ads** | слайдер | набор | свайп |
| **Native ad cards** | лента/related | карточка‑«спонсор» | адаптив |
| **Video ads** | плеер | pre/mid/post‑roll, VAST/VPAID | адаптив |
| **In‑image / rich media** | над изображением | overlay | адаптив |

## 10.3. Параметры каждого рекламного блока (поля в CMS)

- **Название** и внутренний код (zone key).
- **Место размещения** (страница/шаблон/позиция, привязка к слоту).
- **Размеры** (набор форматов, responsive‑маппинг по брейкпоинтам).
- **Адаптивность** (какой формат на каком разрешении).
- **Период показа** (start/end дата‑время), **расписание** (дни недели, часы, таймзона).
- **Ограничения:** макс. показов всего / в день, макс. кликов, бюджет.
- **Приоритет** (guaranteed/house/network) и **вес показа** (weight для ротации).
- **Таргетинг:** страны, города/регионы (гео‑IP), языки, устройства (desktop/mobile/tablet), ОС, браузеры, рубрики/теги статьи, тип пользователя (гость/подписчик — напр. не показывать подписчику), новые/вернувшиеся, UTM‑источник, время суток, referrer.
- **Частота (frequency capping):** N показов на пользователя за период (cookie/ID).
- **A/B тестирование:** несколько креативов, split %, авто‑победитель по CTR/CR.
- **Статистика:** показы, viewable‑показы, клики, **CTR**, конверсии, переходы, доход (CPM/CPC/CPA), viewability %, по срезам (день/гео/устройство/страница).
- **Экспорт отчётов:** CSV/XLSX/PDF, планировщик отчётов на e‑mail.

## 10.4. Сущности

- **Advertiser:** название, контакты, договор, баланс/бюджет.
- **Campaign:** цель, тип покупки (CPM/CPC/CPA/flat), бюджет, флайт (даты), статус.
- **Creative:** формат, файлы (изображение/HTML5/видео/VAST‑URL), клик‑URL + UTM, alt, вес, A/B‑вариант.
- **Ad Slot/Zone:** позиция, допустимые форматы, fallback (house‑ad/пусто), lazy‑настройки.
- **House ads:** собственные промо (подписка, приложения) при отсутствии оплаченной рекламы.

## 10.5. Внешние сети и интеграции

Единый раздел «Интеграции → Реклама»: **Google Ad Manager (GAM)**, **Google AdSense**, **Google Ads**, **Header Bidding (Prebid.js)**, programmatic (open/private auctions), прямые сделки. Настройка: ad‑unit ID, размеры, lazy‑load, consent (TCF/GDPR), waterfall/приоритет между собственным сервером и сетями.

## 10.6. Пиксели и трекинг‑платформы

**Meta Pixel, Google Analytics 4, Google Tag Manager, Yandex Metrica, TikTok Pixel**, LinkedIn Insight, Twitter Pixel — подключаются через раздел интеграций (вставка ID, а не кода), с учётом Consent Mode (не грузить до согласия). События: page_view, article_read, scroll_depth, subscribe_start/complete, ad_impression/click, video_play, newsletter_signup.

## 10.7. Управляемость владельцем

- Визуальный менеджер: создать рекламодателя → кампанию → загрузить креатив → выбрать зону → задать таргетинг/расписание/частоту → запустить → смотреть статистику.
- Preview‑режим (как выглядит на странице).
- Автопауза при исчерпании бюджета/срока.
- Уведомления (кампания заканчивается, низкий CTR, отклонён креатив).
- Роль **Ad Manager** без доступа к контенту.

# 11. SEO‑АРХИТЕКТУРА (ЭТАП 7)

## 11.1. Мета и разметка

- **Meta Title:** шаблон `{Заголовок} — {Рубрика} | {Название издания}`, лимит ~60 симв., переопределяется вручную.
- **Meta Description:** авто из dek/первого абзаца (~150–160 симв.), редактируемая.
- **Keywords:** опционально (низкий вес), из тегов.
- **Open Graph:** og:title, og:description, og:image (1200×630), og:type=article, og:url, og:locale, article:published_time, article:author, article:section, article:tag.
- **Twitter Cards:** summary_large_image, twitter:site/creator/title/description/image.
- **Canonical:** абсолютный URL, авто; правила для пагинации/фильтров (canonical на базовую), AMP‑связь (если используется).
- **Robots meta:** index/noindex, follow/nofollow (черновики/фильтры/пагинация — noindex), max‑image‑preview:large.
- **hreflang:** для мультиязычных версий (uz/ru/en) + x‑default.

## 11.2. Структурные данные (Schema.org / JSON‑LD)

- **NewsArticle / Article** (headline, image, datePublished, dateModified, author, publisher, articleSection, wordCount, isAccessibleForFree + hasPart для paywall — критично для Google News/индексации premium).
- **Organization / NewsMediaOrganization**, **WebSite** (+ SearchAction — sitelinks searchbox), **BreadcrumbList**, **VideoObject**, **PodcastEpisode**, **FAQPage**, **Person** (авторы), **ImageObject**.
- Paywall‑разметка `isAccessibleForFree:false` + CSS‑селектор платной части (Google flexible sampling).

## 11.3. URL / ЧПУ

- Человекопонятные: `/[section]/[slug]`, транслитерация кириллицы/узбекского, нижний регистр, дефисы, без stop‑words, стабильные (при смене заголовка slug не меняется без 301).
- Языковой префикс: `/ru/…`, `/uz/…`, `/en/…`.

## 11.4. Sitemap и индексация

- XML‑sitemap с разбиением (sitemap‑index): статьи, рубрики, видео‑sitemap, image‑sitemap, **News sitemap** (Google News, последние 48 ч), авторы, статические.
- Авто‑генерация и пинг поисковиков при публикации.
- robots.txt (управляемый из CMS), IndexNow/Google Indexing API для быстрой индексации.

## 11.5. Прочее

- **Редиректы:** 301 при смене slug/удалении, менеджер редиректов, регэксп‑правила, лог 404 → предложение редиректа.
- **Хлебные крошки** (визуальные + BreadcrumbList).
- **Alt‑тексты** обязательны, авто‑подсказки (ML), контроль в медиа‑библиотеке.
- **Микроразметка** тикеров/цен, дат, авторов.
- **Core Web Vitals** как ранжирующий фактор (см. Производительность).
- **RSS/Atom** фиды по рубрикам, **Google News** совместимость, **AMP** (опц., по решению).

# 13. REST API (ЭТАП 9)

**База:** `/api/v1`, JSON, версионирование в URL. Авторизация: **JWT (Bearer)** + refresh, для админ — сессии/JWT с RBAC. Публичное чтение — по API‑ключу/анонимно с rate‑limit.

## 13.1. Соглашения

- **Формат ответа:**
```json
{ "data": {...|[...]}, "meta": { "page":1, "per_page":20, "total":345 }, "links": { "next":"...", "prev":"..." } }
```
- **Ошибки:**
```json
{ "error": { "code":"VALIDATION_ERROR", "message":"...", "fields": { "email":["Обязательное поле"] } } }
```
- **Коды:** 200 OK, 201 Created, 204 No Content, 301/302 redirect, 400 Bad Request, 401 Unauthorized, 403 Forbidden (нет прав/paywall), 404 Not Found, 409 Conflict (дубль slug), 410 Gone (снято), 422 Validation, 429 Too Many Requests, 451 (правовая блокировка), 500/503.
- **Пагинация:** cursor/offset; **фильтры/сортировка** через query (`?section=markets&sort=-published_at&filter[type]=video`).
- Идемпотентность (Idempotency‑Key) для POST платежей; ETag/If‑None‑Match для кэша.
