# Подключение внешних сервисов (Google / Яндекс и др.)

Всё активируется через переменные окружения — впишите значения в `.env` (или в настройки хостинга) и перезапустите/пересоберите сайт. Пример всех переменных — в `.env.example`.

> Важно: переменные с префиксом `NEXT_PUBLIC_` попадают в браузер и **встраиваются на этапе сборки** — после их изменения нужно пересобрать (`npm run build`) или перезапустить `npm run dev`.

---

## 1. Верификация в поисковиках (обязательно для индексации и рекламы)

### Google Search Console
1. Откройте https://search.google.com/search-console → **Добавить ресурс** → «Ресурс с префиксом URL» → ваш домен.
2. Способ подтверждения → **HTML-тег**. Скопируйте значение `content="..."`.
3. Впишите в `.env`: `NEXT_PUBLIC_GOOGLE_SITE_VERIFICATION=скопированное_значение`
4. Пересоберите сайт → нажмите «Подтвердить». Мета-тег уже отдаётся автоматически.
5. Там же добавьте **sitemap**: `https://ваш-домен/sitemap.xml`.

### Яндекс.Вебмастер
1. https://webmaster.yandex.ru → **Добавить сайт** → ваш домен.
2. Способ → **Мета-тег**. Скопируйте `content="..."`.
3. `.env`: `NEXT_PUBLIC_YANDEX_VERIFICATION=значение` → пересобрать → «Проверить».
4. Добавьте sitemap там же.

Robots.txt и sitemap.xml уже отдаются сайтом (`/robots.txt`, `/sitemap.xml`) и разрешают Googlebot/YandexBot.

## 2. Аналитика

| Сервис | Где взять ID | Переменная |
|---|---|---|
| **Google Analytics 4** | https://analytics.google.com → Администратор → Потоки данных → «Идентификатор потока» `G-XXXXXXX` | `NEXT_PUBLIC_GA_ID` |
| **Google Tag Manager** | https://tagmanager.google.com → создать контейнер → `GTM-XXXXXX` | `NEXT_PUBLIC_GTM_ID` |
| **Яндекс.Метрика** | https://metrika.yandex.ru → добавить счётчик → числовой номер | `NEXT_PUBLIC_YM_ID` |

Скрипты подключаются автоматически, только если ID задан. Метрика включена с вебвизором и картой кликов.

## 3. Реклама

### Google AdSense
1. https://adsense.google.com → добавить сайт → пройти проверку (нужна верификация из п.1 и наполнение).
2. AdSense выдаст строку для **ads.txt** вида: `google.com, pub-0000000000000000, DIRECT, f08c47fec0942fa0`.
3. `.env`: `ADS_TXT="google.com, pub-0000..., DIRECT, f08c47fec0942fa0"` (несколько строк — через `\n`).
4. Файл отдаётся по адресу `https://ваш-домен/ads.txt`.
5. Рекламные блоки: вставляются в компонент показа рекламы (`AdSlot`) — дайте код блока AdSense, подключу.

### Яндекс.Директ / РСЯ
1. https://partner.yandex.ru (Рекламная сеть Яндекса) → подключить площадку.
2. Строку для `ads.txt` от РСЯ добавьте туда же (в `ADS_TXT`, с новой строки).
3. Код рекламного блока РСЯ — вставим в `AdSlot`.

## 4. reCAPTCHA (защита форм входа/регистрации)
1. https://www.google.com/recaptcha/admin → **+** → тип **reCAPTCHA v2** → «Я не робот».
2. Домены: ваш домен (+ `localhost` для тестов).
3. Скопируйте **Site Key** и **Secret Key**.
4. `.env`:
   ```
   NEXT_PUBLIC_RECAPTCHA_SITE_KEY=site_key
   RECAPTCHA_SECRET_KEY=secret_key
   ```
Сейчас стоят тестовые ключи Google (пропускают всех) — замените на боевые.

---

## 5. Почта (сброс пароля, подтверждение e-mail)
1. https://resend.com → зарегистрироваться → **API Keys** → создать ключ.
2. Подтвердите свой домен в Resend (Domains) — иначе письма шлются только на свой адрес.
3. `.env`:
   ```
   RESEND_API_KEY=re_xxx
   EMAIL_FROM="Asosiy Aktiv <noreply@ваш-домен>"
   ```
Без ключа поток работает в dev-режиме (токен в логе). С ключом письма уходят реально, а токен в ответе больше не отдаётся.

---

## 6. OAuth-вход (Google / Apple) — по запросу
- Google: https://console.cloud.google.com → «Учётные данные» → OAuth client ID (Web) → redirect `https://ваш-домен/api/auth/callback/google`.
- Понадобятся `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET`. Скажите — подключу поток (кнопки «Войти через Google» сейчас заглушки).

---

## 7. Хранилище медиа (для облака)
Сейчас загрузки сохраняются на диск в `public/uploads` (работает для self-hosted / VPS). Для Vercel/облака нужен внешний бакет (S3, Cloudflare R2, Vercel Blob) — интерфейс загрузки (`/api/upload`) уже единый, подставим SDK бакета по вашим ключам.

---

## 8. Приём оплат (тарифы компаний) — по запросу
Payme / Click / Uzum / Stripe — нужен договор/аккаунт мерчанта и ключи. После этого подключим приём платежей и генерацию инвойсов на странице «Финансы».
